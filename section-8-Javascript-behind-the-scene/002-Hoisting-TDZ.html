Javascript Hoisting and Temporal Dead Zone(TDZ)

variable declared with var is hoisted and it's default value is "undefined", when you try to access its before declaration we will get "undefined" that's the value assigned before execution of the code line by line.


When we try to access variable created using 'let' and 'const', we will get RefernceError saying that 'cannot access "variable" before initialization', before before execution of code it's default value assigned is "uninitialized" - they are not hoisted, they are placed inside TDZ(temporal dead zone), they are block scoped 

let and const even though declared in global variable are not a part of window object
      var x = 1;
      let y = 20;
      const z = 29;

      console.log(x === window.x); //true
      console.log(y === window.y); //false
      console.log(z === window.z); //false
